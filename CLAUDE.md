# CLAUDE.md

このファイルは、Claude Code がこのディレクトリで作業する際のガイダンスを提供する。

本文書の【MUST】【SHOULD】【MAY】は、[RFC 2119](https://datatracker.ietf.org/doc/html/rfc2119)に準拠：
- 【MUST GLOBAL】：MUSTの上位概念。プロジェクトごとのCLAUDE.mdより優先される
- 【MUST】: 絶対的要求事項。例外なく従う必要がある
- 【SHOULD】: 強い推奨事項。特別な理由がない限り従う
- 【MAY】: 任意事項。状況に応じて採用を判断

## 【MUST GLOBAL】大前提

- 日本語で応答する（コード・変数名は英語）
- 簡潔に回答し、自明な説明は省略する

## コーディングガイドライン

速度よりも慎重さを重視する。些細なタスクでは適宜判断してよい。

### 【MUST GLOBAL】1. コードを書く前に考える

- 前提を明示的に述べる。不確かなら質問する
- 複数の解釈がありえる場合、黙って選ばず提示する
- よりシンプルな方法があるなら提案する。必要なら反論する
- 不明点があれば止まる。何が不明かを名指しして質問する

### 【MUST GLOBAL】2. シンプルさ最優先（KISS の原則を守れ）

- 求められていない機能を追加しない
- 一度しか使わないコードに抽象化を持ち込まない
- 要求されていない「柔軟性」「設定可能性」を入れない
- 起こりえないシナリオのエラーハンドリングを書かない
- 200行で書いたものが50行でできるなら書き直す
- 判断基準: シニアエンジニアが「過剰に複雑だ」と言うならば簡素化する

### 【MUST GLOBAL】3. 外科的な変更

- 必要な箇所だけ触る
- 変更した全行がユーザーのリクエストに直接紐づくこと
- ファイル削除時は `*` を使わずフルパスで `rm` を実行する

既存コード編集時:
- 隣接するコード・コメント・フォーマットを「改善」しない
- 壊れていないものをリファクタリングしない
- 自分のスタイルと違っても既存スタイルに合わせる
- 無関係なデッドコードに気づいたら、削除せず指摘だけする

自分の変更で生じた不要物の処理:
- 自分の変更で不要になったインポート・変数・関数 → 削除する
- 元から存在していたデッドコード → 依頼されない限り削除しない

レビュー:
- 小さい変更でなければ、実装したら Subagent でレビューせよ（可能であれば複数の観点で）

### 【MUST GLOBAL】4. ゴール駆動の実行

- タスクを検証可能なゴールに変換してから着手する
  - 「バリデーション追加」→「不正入力のテストを書き、パスさせる」
  - 「バグ修正」→「再現テストを書き、パスさせる」
  - 「Xをリファクタ」→「リファクタ前後でテストがパスすることを確認」
- 複数ステップのタスクでは `[ステップ] → 検証: [チェック内容]` の形式で計画を示す
- 明確な成功基準があれば自律的にループする
- 曖昧な基準（「動くようにして」）は都度ユーザーに確認する

## 【SHOULD】Plan / Spec の書き出し

- 小さい変更であれば必ずしも必要ない
- 実装計画（plan）は `~/Documents/Obsidian/Vault/Plans/` に Markdown で作成する
- 仕様書（spec）は `~/Documents/Obsidian/Vault/Specs/` に Markdown で作成する
- ファイル名は内容がわかる日本語名にする（例: `認証リファクタ計画.md`）
- カレントディレクトリやプロジェクト内に plan/spec ファイルを作らない
- 小さい変更でなければ、作成したら Subagent でレビューせよ（可能であれば複数の観点で）

## 【SHOULD】作業ログ

- 作業が一区切りついたら、その都度作業ログを残す。区切りの目安は次の通り:
  - タスクが完了したとき
  - まとまった単位（コミット・PRマージ・調査など）が一段落したとき
  - ユーザーがセッションを切り上げる素振りを見せたとき
- 「後で書く」はしない。会話の流れの中でその場に残す
- 出力先は `~/Documents/Obsidian/Vault/メモ/YYYY-MM-DD.md`（`worklog` スキル相当。既存があれば末尾に追記）
- 実際にやった作業だけを簡潔に。推測で埋めない。コミットハッシュや関連ノートの [[link]] を添える

## 【SHOULD】CLAUDE.md の同期

- `~/.claude/CLAUDE.md` は `~/repos/claude-md/CLAUDE.md` へのシンボリックリンク
- ローカル固有セクション（スコープ・フック等）はプロジェクト別の CLAUDE.md に分離すること

## 【SHOULD】Git 規約

- [Conventional Commits](https://www.conventionalcommits.org/en/v1.0.0/) 形式（`feat:`, `fix:`, `docs:` 等）、本文は日本語
- 確認なしに自動コミット・自動 push しない
